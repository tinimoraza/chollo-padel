## ERRORES CORREGIDOS (commits subidos)

### Error 1 — audit-matches/route.ts (commit 75f35fb)
Fichero: app/api/cron/audit-matches/route.ts
Error:   Type error: Property 'id' does not exist on type 'ParserError<"Unexpected input: ño">'
Causa:   TypeScript no infería bien los tipos de Supabase en columnas con ñ ("año")
Fix:     Añadido cast `as unknown as { data: { id, marca, modelo, año }[] | null }`
         en las dos queries a tabla `palas` (líneas ~169 y ~281)

### Error 2 — match-wallapop/route.ts (commit 45b019e)
Fichero: app/api/cron/match-wallapop/route.ts
Error:   Property 'catch' does not exist on type 'PostgrestFilterBuilder...'
Causa:   .catch() no funciona encadenado en queries de Supabase v2
Fix:     Reemplazados los dos .catch(() => {}) por bloques try/catch con await

### Error 3 — match-pala-id.ts (commit en curso)
Fichero: scripts/match-pala-id.ts
Error:   An object literal cannot have multiple properties with the same name ('vibora')
Causa:   Clave 'vibora' duplicada en MARCA_MAP (línea 194), ya existía 'vibor-a'
Fix:     Eliminada la línea duplicada 'vibora': 'Vibora'

### Error 4 — match-pala-id.ts (mismo fichero, error anterior)
Fix previo en este fichero: cambiada la firma de matchPalaIds para aceptar
